# README

## HEMS App 기능명세서 안내서

본 문서는 `11_HEMS_기능명세서 - App_기능명세서.csv` 파일을 **README.md 형태로 활용**하기 위한 공식 가이드 문서이다.
GitHub/GitLab 등 형상관리 저장소에서 기능명세서를 함께 관리할 때, 본 문서를 **최상위 README.md**로 사용한다.

---

## 1. 문서 목적

이 README는 다음을 목표로 한다.

* HEMS App 기능명세서 CSV의 **구조와 해석 방법을 명확히 설명**한다.
* 신규 참여자(기획/개발/QA/운영)가 **짧은 시간 안에 전체 기능 체계를 이해**하도록 돕는다.
* 기능명세서 → 상세 기능정의서 → 화면기획 → QA 산출물 간의 **공통 기준(Code)** 을 제공한다.

---

## 2. 문서 범위

* 대상 시스템: **HEMS (Home Energy Management System) App**
* 포함 범위: 사용자(App) 기능 정의
* 제외 범위:

  * Admin 기능
  * API/DB 설계
  * UI 상세 시안

본 문서는 **제품 기획 관점의 기능 정의 기준**만을 다룬다.

---

## 3. 기능명세서 CSV 파일

### 파일명

```
11_HEMS_기능명세서 - App_기능명세서.csv
```

### 역할

* HEMS App의 **모든 기능에 대한 단일 기준(Source of Truth)**
* 모든 하위 산출물은 본 CSV의 Code를 기준으로 연결된다.

---

## 4. CSV 컬럼 구조 설명

| 컬럼명         | 설명                      |
| ----------- | ----------------------- |
| Group       | 기능의 상위 분류               |
| Code        | 기능 고유 식별자 (Primary Key) |
| Title       | 기능명                     |
| Description | 기능 요약 설명                |
| 요구사항 코드     | 상위 요구사항(SFR 등)과의 연결     |
| 비고          | 보완 설명 또는 참고 사항          |
| H-Cloud 수신  | H-Cloud → App 데이터       |
| H-Cloud 송신  | App → H-Cloud 데이터       |
| 자체처리        | App/로컬에서 자체 처리하는 로직     |

---

## 5. Group 분류 체계

| Group     | 설명                |
| --------- | ----------------- |
| COM       | 공통 규칙 및 공통 데이터 처리 |
| NOW       | 현재 상태 / 실시간 정보 표시 |
| DAT       | 데이터 분석 및 지표 산출    |
| SET       | 사용자 설정 기능         |
| VAL       | 예측·시뮬레이션·평가 기능    |
| NOT       | 알림 및 이벤트 기능       |
| MEM / HIS | 설정·제어 이력 및 로그     |

> Group은 화면 기준이 아닌 **기능 성격 기준 분류**이다.

---

## 6. Code 규칙

* Code 형식: `GROUP_번호`
* Group 내에서 **번호는 유일**하다.
* Code는 절대 변경하지 않으며, 기능의 영구 식별자로 사용한다.

### 예시

* `COM_001` : 공통 영역 기능
* `NOW_006` : 실시간 상태 표시 기능

### Code 사용처

* 상세 기능정의서
* 화면 기획서
* QA 시나리오
* 개발 이슈 및 테스트 케이스

---

## 7. Description 해석 가이드

Description에는 다음 정보가 포함될 수 있다.

* 기능의 목적
* 제공 데이터 범위
* 계산/표현에 대한 전제 조건
* 정책적 또는 스펙 제약

> 계산식, 수치 기준, 예외 조건은 **상세 기능정의서**에서 정의한다.

---

## 8. H-Cloud 연계 컬럼 가이드

| 컬럼         | 의미                      |
| ---------- | ----------------------- |
| H-Cloud 수신 | 외부(H-Cloud)에서 전달되는 데이터  |
| H-Cloud 송신 | App에서 H-Cloud로 전달하는 데이터 |
| 자체처리       | 외부 연계 없이 내부 처리          |

해당 컬럼은 **아키텍처 책임 구분을 위한 참고 정보**이며,
기능 존재 여부를 판단하는 기준은 아니다.

---

## 9. 사용 원칙

1. 기능 정의의 기준은 항상 본 CSV이다.
2. 기능 추가/변경은 반드시 Code 단위로 관리한다.
3. 상세 기능정의서는 CSV의 Code와 1:1로 매핑한다.
4. 기존 기능은 삭제하지 않고 확장 또는 Deprecated 처리한다.
