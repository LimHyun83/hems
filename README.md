# HEMS 기능명세서 통합 README

본 문서는 **HEMS(Home Energy Management System)** 프로젝트의
**App / Admin 기능명세서(CSV)를 함께 분석·운영하기 위한 GitHub용 공식 README.md**이다.

이 저장소에서 관리되는 모든 기능 정의, 상세 문서, QA 기준은
본 README와 각 CSV 기능명세서를 **단일 기준(Source of Truth)** 으로 사용한다.

---

## 1. 문서 목적

이 README는 다음을 목적으로 한다.

* App / Admin 기능명세서 CSV의 **역할과 경계를 명확히 정의**한다.
* CSV → Markdown → 화면/QA로 이어지는 **문서 체계의 기준을 제공**한다.
* 신규 참여자(기획/개발/QA/운영)가 **빠르게 전체 구조를 이해**하도록 돕는다.

---

## 2. 시스템 구성 개요

HEMS는 다음 두 개의 주요 영역으로 구성된다.

| 구분    | 설명                                   |
| ----- | ------------------------------------ |
| App   | 최종 사용자(Home Owner)를 위한 에너지 관리 애플리케이션 |
| Admin | 운영자/관리자를 위한 설정·모니터링·정책 관리 도구         |

App과 Admin은 **기능 목적·사용자·책임이 명확히 분리**되며,
각각 독립된 기능명세서를 가진다.

---

## 3. 기능명세서 CSV 구성

### 3.1 App 기능명세서

```
HEMS_기능명세서 - App_기능명세서.csv
```

* 사용자 대상 기능 정의
* 에너지 현황 조회, 분석, 설정, 알림 제공
* 사용자 경험(UX) 기준의 기능 단위 정의

### 3.2 Admin 기능명세서

```
HEMS_기능명세서 - Admin_기능명세서.csv
```

* 운영자/관리자 대상 기능 정의
* 시스템 설정, 정책 관리, 장치·계정 관리
* 운영·검증 관점의 기능 단위 정의

> 두 CSV는 **서로 대체 관계가 아니며**, 함께 전체 HEMS 기능을 완성한다.

---

## 4. CSV 공통 구조 개념

App / Admin CSV는 동일한 개념 구조를 따른다.

| 항목          | 설명                |
| ----------- | ----------------- |
| Group       | 기능의 상위 분류         |
| Code        | 기능 고유 식별자 (영구 ID) |
| Title       | 기능명               |
| Description | 기능 요약 설명          |
| 비고          | 제약 또는 참고 사항       |

Code는 모든 문서와 산출물을 연결하는 **단일 기준 키**이다.

---

## 5. Group 개념 (공통)

Group은 화면이나 메뉴가 아닌 **기능 성격 기준 분류**이다.

| Group     | 의미             |
| --------- | -------------- |
| COM       | 공통 규칙 및 공통 데이터 |
| NOW       | 현재 상태 / 실시간 정보 |
| DAT       | 데이터 조회 및 분석    |
| SET       | 설정 기능          |
| VAL       | 예측·시뮬레이션·평가    |
| NOT       | 알림 및 이벤트       |
| MEM / HIS | 이력·로그          |

Admin CSV에는 App에 없는 **운영 전용 Group**이 포함될 수 있다.

---

## 6. App vs Admin 기능 경계 원칙

### App 기능의 성격

* 사용자가 직접 인지·조작하는 기능
* 결과 중심(조회, 분석, 안내)
* UX 관점의 기능 단위

### Admin 기능의 성격

* 시스템을 **설정·통제·검증**하기 위한 기능
* 정책·기준·권한 중심
* 운영 안정성 관점의 기능 단위

> 동일한 개념의 기능이라도 **사용자 주체가 다르면 Code는 분리**한다.

---

## 7. Markdown 문서 체계

CSV는 목록 관리에 적합하고,
Markdown은 설명·리뷰·합의에 적합하다.

### 권장 디렉터리 구조

```text
/HEMS
 ├─ README.md              # 본 문서
 ├─ product
 │   ├─ README.md          # App/Admin 기능명세서 가이드
 │   └─ spec/
 │       ├─ COM/
 │       ├─ NOW/
 │       ├─ DAT/
 │       └─ ...
 ├─ csv/
 │   ├─ HEMS_기능명세서 - App_기능명세서.csv
 │   └─ HEMS_기능명세서 - Admin_기능명세서.csv
 └─ changelog/
     └─ CHANGELOG.md
```

---

## 8. CSV ↔ Markdown 매핑 원칙

* CSV의 **1 Code = 1 Markdown 파일**
* Markdown은 CSV를 설명·확장할 뿐, 의미를 변경하지 않는다.
* CSV 변경 시 관련 Markdown을 반드시 동기화한다.
